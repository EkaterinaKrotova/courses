**Первая часть** задачи состоит в том, чтобы запустить контейнер с приложением в трёх репликах через `docker-compose`, дабы через nginx контейнер достучаться до него по имени docker-compose сервиса `app`, таким образом в перспективе балансировав трафик между контейнерами с приложением через nginx

Предварительно, необходимо создать Docker сеть с именем `mynetwork` через команду `docker network create mynetwork`

Сделать это можно используя флаг `-p (--project-name)` следующими командами:
- `docker-compose -p dev up -d app`
- `docker-compose -p qa up -d app`
- `docker-compose -p prod up -d app`
- `docker-compose up -d nginx`

**Вторая часть** задачи состоит в том, чтобы несколько раз пересоздать контейнеры и их приватные IP-адреса изменились.

**Почему вторая часть задачи важна:** в какой-то момент, обращаясь по DNS адресу сервиса docker compose (в нашем случае, `app`) после перезапуска контейнеров с приложением, nginx может кидать на один из вновь запущенных контейнеров фразу `connection refused/timeout` из-за того, что DNS имя не поменялось, но при этом целевой IP-адрес контейнера изменился, но, в свою очередь, nginx этого не увидел

**Итоговая цель:** получить эту заветную фразу при обращении к nginx (например, выполняя `curl localhost` на хосте)
